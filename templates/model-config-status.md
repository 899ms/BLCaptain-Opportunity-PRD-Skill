# 模型配置状态

| 字段 | 内容 |
|---|---|
| 配置状态 | config_required / low_confidence / standard / heavy_discussion |
| 已配置模型数 | 0 |
| 健康检查通过数 | 0 |
| Codex 主持状态 | 可主持 / 需配置模型 |
| 降级说明 | 无可用模型时只输出配置引导 |

## 三步配置引导

1. 添加模型：填写模型名称，例如 Model A、Model B、Local Model。
2. 填调用方式：环境变量、OpenAI-compatible URL、CLI 命令或内置能力。
3. 测试并选择用途：长文本、商业反方、结构化、外部趋势、代码实现或通用。

## 继续条件

- 通过至少 1 个模型健康检查后，才进入机会分析。
- 只有 1 个模型时，标注单模型低置信度。
