# 方法论路由

## 原则

方法论是判断工具，不是填表任务。每次只选择能影响 Gate、MVP 或 PRD 边界的方法，默认 2 到 4 个。

## 路由表

| 触发条件 | 方法 | 输出结论 |
|---|---|---|
| 输入模糊 | 6W2H | 用户、场景、动作、频次、成本、限制 |
| 原话多但原因散 | 5Why + JTBD | 根因和用户真正要完成的任务 |
| 需要产品感 | NABC | Need、Approach、Benefit、Competition |
| 需要商业判断 | ROI + 商业画布关键格 | 成本节省、付费锚点、渠道、单位经济 |
| 功能超过 3 个 | KANO | P0、P1、删除项 |
| 外部环境影响大 | PEST | 政策、经济、社会、技术风险 |
| 需要拆执行 | 拆推评算 | 拆对象、推路径、评价值、算成本 |
| 需要结构化沟通 | MECE + 金字塔 | 分类不重不漏，结论先行 |

## 输出格式

```markdown
## 方法论选择

| 方法 | 触发原因 | 关键结论 | 对 MVP 的影响 |
|---|---|---|---|
```

## 禁止项

- 不机械套用所有方法。
- 不输出只有方法名称、没有结论的章节。
- 不让方法论结论覆盖证据不足。
- 方法超过 4 个时，删除对 Gate 没有影响的方法。
