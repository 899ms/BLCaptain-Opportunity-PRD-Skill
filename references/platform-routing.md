# 动态平台路由

## 目标

根据机会类型选择社区组合，不固定平台，不追求抓取数量，优先追求真实用户原话和手动行为。

## 路由维度

| 维度 | 例子 | 平台候选 |
|---|---|---|
| 开发者工具 | API、IDE、CLI、开源库 | GitHub Issues、Hacker News、V2EX、Reddit、Stack Overflow |
| B2B SaaS | CRM、客服、销售、财务、协作 | G2、Capterra、Product Hunt、Indie Hackers、LinkedIn、X |
| 电商和运营 | 店铺运营、客服、投放、内容 | 小红书、知乎、抖音评论、商家论坛、Reddit smallbusiness |
| 消费生活 | 健康、教育、家庭、兴趣 | 小红书、知乎、豆瓣、Reddit 垂直版、应用商店评论 |
| AI 工具 | 提示词、自动化、生成式内容 | Product Hunt、X、Reddit、即刻、V2EX、GitHub |

## 输出要求

平台路由必须包含：

- 平台名称
- 选择理由
- 搜索关键词或查询式
- 预期信号
- 获取方式：公开搜索、站内搜索、用户粘贴样本、API
- 降级方案

## 数据获取降级

| 层级 | 方式 | 置信度 |
|---|---|---|
| L1 | API、公开接口、可验证页面 | 高 |
| L2 | 搜索引擎、站内搜索、公开页面摘录 | 中 |
| L3 | 用户粘贴评论样本 | 取决于样本完整度 |

## 禁止项

- 不因为某个平台热门就默认选择。
- 不把平台上的文章摘要当作评论证据。
- 不用单一社区支持 Go。
