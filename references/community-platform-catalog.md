# 社区平台候选目录

## 原则

本 skill 不固定抓取平台，也不默认全量抓取。每次任务根据产品想法、用户画像、行业、语言、竞品和证据缺口，从候选库里路由 5 到 12 个入口。

平台只提供线索入口，不自动代表有效证据。有效证据仍必须满足：近 30 天、用户原话、日期、URL 或可追溯来源、行为信号、商业信号和反向证据。

## 获取方式分层

| 层级 | 方式 | 说明 |
|---|---|---|
| L1 | 公开 URL、公开 API、站内可验证页面 | 优先使用，可复查 |
| L2 | 搜索引擎、公开摘录、聚合页 | 需回到原始评论验证 |
| L3 | 用户粘贴样本、本地导出文件 | 可用，但需保留来源和采样说明 |
| L4 | 登录态、私密群、付费后台 | 不自动抓取，只能由用户授权后提供样本或 API |

## 当前样例已覆盖

## 路由输出格式

每次生成平台路由时，至少输出：

- 平台名称
- 选择理由
- 查询式或关键词
- 预期信号
- 获取方式
- 降级方案
- 是否需要用户授权或粘贴样本
