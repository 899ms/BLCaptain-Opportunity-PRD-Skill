# G0 到 G8 Gate 规则

Gate 是本 skill 的否决系统。未通过 Gate 时，不得继续生成更确定的下游产物。

| Gate | 通过阈值 | 否决阈值 | 失败后动作 |
|---|---|---|---|
| G0 多模型配置 | 至少 1 个外部模型通过健康检查，Codex 可主持 | 无可用外部模型 | 只输出配置引导 |
| G1 意图消歧 | 至少明确目标用户、场景、商业目标中的 2 个 | 三者都缺失 | 向用户提最多 3 个问题 |
| G2 平台路由 | 至少 3 类平台，每类有选择理由 | 只选 1 个平台且无理由 | 重做路由 |
| G3 证据墙 | 至少 5 条有效证据，来自至少 2 个独立社区 | 有效证据少于 3 条 | 输出证据不足报告 |
| G4 数据可信度 | 有效样本数 / 原始样本数 >= 30% | 低于 15% | 标注低置信度，不进入 PRD |
| G5 商业信号 | 至少 1 条付费、弃用竞品、成本节省或购买意图证据 | 完全无商业信号 | 输出 Watch |
| G6 反向证据 | 至少 3 条反向证据并逐条回应 | 存在无法回应的合规或免费替代反证 | Pivot 或 No-Go |
| G7 MVP 可验证 | 7 天内能测试 1 个关键假设，且有明确用户来源 | 无法找到测试用户 | 收窄切口 |
| G8 PRD 可交接 | P0 功能 100% 绑定 evidence_id，且有 3 条验收剧本 | 任一 P0 无证据 | 修复 PRD 或删除该功能 |

## 决策

| 决策 | 条件 | 输出 |
|---|---|---|
| Go | G0 到 G8 均通过或低风险项有明确降级说明 | 机会评估报告 + 商业化机会 PRD |
| Watch | 痛点存在，但商业信号或证据强度不足 | 机会评估报告 + 补证计划 |
| Pivot | 原方向不成立，但旁支机会成立 | 重新定义切口 |
| No-Go | 反证压倒正证，或核心 Gate 被否决 | 停止原因 + 复盘记录 |

## 审计问题

- 每个结论是否引用 evidence_id。
- 每条 evidence_id 是否有原话、URL、日期和平台。
- P0 是否只有 1 到 3 个用户故事。
- 是否存在无证据的“应该做”。
- 是否把模型判断当成真实用户证据。
