# G0 到 G8 Gate 规则

Gate 是本 skill 的否决系统。未通过 Gate 时，不得继续生成更确定的下游产物。

| Gate | 通过阈值 | 否决阈值 | 失败后动作 |
|---|---|---|---|
| G0 多模型配置 | 至少 1 个外部模型通过健康检查，Codex 可主持 | 无可用外部模型 | 只输出配置引导 |
| G1 意图消歧 | 至少明确目标用户、场景、商业目标中的 2 个 | 三者都缺失 | 向用户提最多 3 个问题 |
| G2 平台路由 | 至少 3 类平台，每类有选择理由 | 只选 1 个平台且无理由 | 重做路由 |
| G3 证据墙 | 至少 5 条有效证据，来自至少 2 个独立社区 | 有效证据少于 3 条 | 输出证据不足报告 |
| G4 数据可信度 | 有效样本数 / 原始样本数 >= 30% | 低于 15% | 标注低置信度，不进入 PRD |
| G5 商业信号 | 至少 1 条付费、弃用竞品、成本节省或购买意图证据 | 完全无商业信号 | 输出 Watch |
| G6 反向证据 | 至少 3 条反向证据并逐条回应 | 存在无法回应的合规或免费替代反证 | Pivot-to-Go 或 No-Go |
| G7 MVP 可验证 | 7 天内能测试 1 个关键假设，且有明确用户来源 | 无法找到测试用户 | 收窄切口 |
| G8 PRD 可交接 | P0 功能 100% 绑定 evidence_id，且有 3 条验收剧本 | 任一 P0 无证据 | 修复 PRD 或删除该功能 |

## 决策

| 决策 | 条件 | 输出 |
|---|---|---|
| Go | G0 到 G8 均通过或低风险项有明确降级说明 | 机会评估报告 + 商业化机会 PRD |
| Watch | 痛点存在，但商业信号或证据强度不足 | 机会评估报告 + 补证计划 |
| Pivot | 原方向不成立，但旁支机会可能成立 | 重新定义切口并重新跑 Gate |
| No-Go | 反证压倒正证，或核心 Gate 被否决 | 停止原因 + 复盘记录 |

## Watch / Pivot 后的 Cut-to-Go

Watch 或 Pivot 不是一句话结束。只要正向评论已经形成具体痛点簇，就必须先判断是否能收窄成一个可验证、可商业化、可工程交付的最小切口。

触发条件：

1. 原始命题包含多个痛点簇，直接写 PRD 会过宽。
2. 某个痛点簇至少有 5 条有效证据。
3. 该痛点簇至少有 1 条商业信号。
4. 该痛点簇来自至少 2 个独立平台或来源。
5. 高风险反证可以通过合规边界、配置边界、成本边界或 ICP 边界回应。

执行动作：

1. 生成 `pain-clusters.md/json`。
2. 选择得分最高的最小切口。
3. 用该切口对应的 evidence_id 和反向证据重新跑 G0 到 G8。
4. 新切口 Go 后生成商业化机会 + 工程实施 PRD。
5. 新切口仍失败时，明确写出不生成 PRD 的 Gate 原因、补证动作和停止条件。

## 审计问题

- 每个结论是否引用 evidence_id。
- 每条 evidence_id 是否有原话、URL、日期和平台。
- P0 是否只有 1 到 3 个用户故事。
- 是否存在无证据的“应该做”。
- 是否把模型判断当成真实用户证据。

## Pivot-to-Go loop

当原切口是 Pivot 时，不得直接生成 PRD，也不得直接结束。必须先判断反证是否能被新切口回应：

1. 提取反证标签，例如隐私合规、预算不足、复杂度过高、免费替代、低频一次性。
2. 生成更小切口，例如合规诊断、配置护航、成本控制、低配置默认报告或更明确 ICP。
3. 用同一批 evidence_id 和 reverse_id 重新跑 G0 到 G8。
4. 新切口 Go 后才生成商业化机会 + 工程实施 PRD。
5. 新切口仍失败时输出 Pivot 或 No-Go，并写清补证动作和停止条件。
