# 八步流程

本 skill 的运行骨架固定为：

```text
调研 -> 分析 -> 计划 -> 开发 -> 验证 -> 测试 -> 审计验收 -> 总结
```

## 每一步含义

| 步骤 | 在机会挖掘中的动作 | 回到调研的触发 |
|---|---|---|
| 调研 | 收集输入、模型配置、社区资料、评论原话、反向证据 | 任一关键字段缺失 |
| 分析 | 多模型讨论、痛点聚类、商业信号判断 | 结论无法引用 evidence_id |
| 计划 | 生成平台路由、关键词、方法论路由、7 天实验 | 计划没有验证和停止线 |
| 开发 | 生成报告、PRD、模板、开发交接 | 发现 P0 无证据 |
| 验证 | 检查模型可用、证据有效、Gate 通过 | Gate 不通过 |
| 测试 | 用成功、失败、边界样例测试输出 | No-Go 样例被误判为 Go |
| 审计验收 | 跑脚本、查证据、查商业 Gate、查 P0 evidence_id | 校验失败 |
| 总结 | 说明结论、取舍、下一步、停止原因 | 用户要求继续补证 |

## 运行纪律

- 调研贯穿每一步。
- 不用方法论掩盖证据不足。
- 每次输出都区分事实、推断和未知项。
- 每次 Go 必须同时有正向证据和反向证据回应。
- 每次 No-Go 必须保留停止原因，方便后续复盘。
